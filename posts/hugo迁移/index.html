HUGO迁移

2021-07-11 [Updated: 2021-07-11] :: shaosy
#杂记 

为什么要迁移？⌗

首先，自然是因为我重装了系统🤣

总之，重装完以后，我的HEXO烂掉了，然后npm也烂掉了。之前就听闻基于go的hugo不错，自然这次就选择了hugo作为blog的新的后端

迁移中所遇到的坑⌗

坑1：HUGO不支持数学公式⌗

是的，你没有看错，HUGO真的不原生支持数学公式！！！

解决方案：在/themes/theme-name/layouts/partials中修改，添加mathjax.html从而添加latax支持

其内容为

<script type="text/javascript"
        async
        src="https://cdn.bootcss.com/mathjax/2.7.3/MathJax.js?config=TeX-AMS-MML_HTMLorMML">
MathJax.Hub.Config({
  tex2jax: {
    inlineMath: [['$','$'], ['\\(','\\)']],
    displayMath: [['$$','$$'], ['\[\[','\]\]']],
    processEscapes: true,
    processEnvironments: true,
    skipTags: ['script', 'noscript', 'style', 'textarea', 'pre'],
    TeX: { equationNumbers: { autoNumber: "AMS" },
         extensions: ["AMSmath.js", "AMSsymbols.js"] }
  }
});

MathJax.Hub.Queue(function() {
    // Fix <code> tags after MathJax finishes running. This is a
    // hack to overcome a shortcoming of Markdown. Discussion at
    // https://github.com/mojombo/jekyll/issues/199
    var all = MathJax.Hub.getAllJax(), i;
    for(i = 0; i < all.length; i += 1) {
        all[i].SourceElement().parentNode.className += ' has-jax';
    }
});
</script>

<style>
code.has-jax {
    font: inherit;
    font-size: 100%;
    background: inherit;
    border: inherit;
    color: #515151;
}
</style>

然后再head.html中加入如下语句

{{ partial "mathjax.html" . }}

即可完成对公式的支持了

坑2：live2D⌗

参考https://github.com/stevenjoezhang/live2d-widget

在layout中引入

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/font-awesome/css/font-awesome.min.css">

然后加入

<script src="https://YOUR_WEBSITE_NAME/live2d/autoload.js"></script>

即可完成伟大的萌娘活动！

但是这就引入了一个新的问题——我怎么把这段JS传到github.io上？

坑3： static⌗

在HUGO的文件目录中，static这个文件夹中内容会一摸一样上传到public中，于是我们可以利用这点，在static中存放images和live2d的源代码，而后调用直接从根目录调用即可

坑4： public⌗

HUGO的public和hexo的略微有些区别。虽然二者均可以用Github，但是hugo的架构是每次hugo产生的文件都将存储于public中，而后直接将这个文件夹中内容push至你的github.io仓库中即可。那么显然的，便会存在git历史记录不统一问题。于是在这里，建议先pull下来，然后除去git目录全部删除，再进行生成、git操作，最后上传。其中，生成的基本上均为HTML文件，模板存放在主题的layout和自己的layout中，均必须有single.html。在这里可以通过自己编写来修改全局，个人认为比hexo的js更加直观
